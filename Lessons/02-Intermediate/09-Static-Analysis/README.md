# Python Static Analysis

Static analysis tools inspect code without running the full program. This lesson
shows how linters, formatters, type checkers, coverage tools, and import
organizers catch problems early.

## Lessons

1. [Static Code Analysis](01_static_analysis.ipynb)

## Supporting Files

- [analyze.py](analyze.py) - sample code for analysis demonstrations.
- [coverage.py](coverage.py) - coverage-related example code.
- [import_sort.py](import_sort.py) - import-ordering example.
- [glitchforge](glitchforge) - a deliberately inconsistent package for trying
  code-quality tools.

## Topics

- Linters such as Pylint and Ruff
- Formatters and import sorting
- Type checking
- Test coverage
- Security and maintainability warnings
