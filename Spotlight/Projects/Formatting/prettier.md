# Prettier

## Project

[https://github.com/prettier/prettier](https://github.com/prettier/prettier)

## What It Is

Prettier is an opinionated code formatter for JavaScript, TypeScript, HTML, CSS, Markdown, JSON, YAML, and several other languages through built-in support and plugins.

Like Black in the Python ecosystem, Prettier is designed to end formatting debates. It parses source code into a syntax tree, discards most of the original layout, and prints the code back out using its own consistent rules.

Prettier is especially common in frontend projects, where a single repository might contain TypeScript, JSX, CSS, JSON configuration, Markdown documentation, and other file types. Using one formatter across all of those files keeps the project visually consistent.

## What It's Used For

- Formatting Frontend Code: Prettier is commonly used for JavaScript, TypeScript, JSX, HTML, CSS, and related web files.
- Formatting Project Files: It can also format JSON, YAML, Markdown, and other non-code files that appear throughout a repository.
- Reducing Review Noise: Automated formatting keeps pull requests focused on logic and behavior instead of whitespace and wrapping.
- Editor Integration: Many teams configure Prettier to run on save so files are formatted continuously while developers work.
- CI Checks: Prettier can run in check mode to verify that files are already formatted.
- Cross-Language Consistency: In projects with many file types, Prettier provides one formatting workflow instead of separate rules for every format.
