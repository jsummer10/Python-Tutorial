# Textual

## Project

[https://github.com/Textualize/textual](https://github.com/Textualize/textual)

## What It Is

Textual is a Python framework for building terminal user interfaces. It lets developers create interactive applications that run in the terminal, with widgets, layout, keyboard interactions, reactive state, styling, and event handling.

Most command-line tools are either simple one-shot commands or full graphical/web applications. Textual fits in the middle: it allows a Python program to feel like an app while still running inside a terminal.

Textual builds on ideas from modern UI frameworks, but keeps the deployment model lightweight. Users can often run a Textual app anywhere they can run a Python terminal program.

## What It's Used For

- Terminal Apps: Textual can build full-screen terminal interfaces for tools, dashboards, browsers, and inspectors.
- Interactive Widgets: It provides widgets for buttons, inputs, trees, tables, logs, tabs, and other interface elements.
- Keyboard-Driven Workflows: Textual works well for applications where fast keyboard navigation matters.
- Developer Dashboards: It can present logs, status, queues, metrics, or workflow state in a richer terminal UI.
- Styling Layouts: Textual uses a CSS-inspired styling system to control spacing, colors, layout, and component appearance.
- Local Tools: Teams can build internal tools without committing to a web frontend or native desktop app.
